# Environment Configuration

## Setup Instructions

### 1. Copy the example file
```bash
cp .env.example .env
```

### 2. Update the values based on your setup

**For Android Emulator:**
```
API_BASE_URL=http://10.0.2.2:8080/api/v1
```

**For iOS Simulator:**
```
API_BASE_URL=http://localhost:8080/api/v1
```

**For Physical Device:**
```
API_BASE_URL=http://192.168.1.100:8080/api/v1
```
*Replace `192.168.1.100` with your computer's actual IP address*

### 3. Finding Your Computer's IP Address

**Windows:**
```bash
ipconfig
# Look for "IPv4 Address" under your active network adapter
```

**macOS/Linux:**
```bash
ifconfig
# Look for "inet" under your active network interface
```

## Current Variables

- **API_BASE_URL**: Backend API endpoint URL
- **API_TIMEOUT**: Request timeout in seconds (default: 30)
- **LOUNGE_OWNER_ID**: Hardcoded owner ID for testing (will be removed when auth is implemented)

## Troubleshooting

### Cannot connect to backend from emulator

1. Make sure backend server is running on `localhost:8080`
2. For Android emulator, verify you're using `10.0.2.2` not `localhost`
3. Check firewall settings allow connections on port 8080

### Cannot connect from physical device

1. Ensure your phone and computer are on the same network
2. Use your computer's local IP address (not localhost or 10.0.2.2)
3. Disable any VPN that might block local network access
4. Make sure backend server is bound to `0.0.0.0` not just `127.0.0.1`
